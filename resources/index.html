Tailwind üì¶

Tailwind CSS is a utility framework that allows you to design responsive and customizable web interfaces by applying classes directly to HTML, without imposing a pre-designed aesthetic. It enables more flexibility to create your site from beginning, but it involves a harder learning-curve and it requires Node.js and a modern development environment.

Smaller CSS

I found it very curious how Tailwind automatically removes unnecessary CSS so the webpage loads faster. I have struggled in the past with big CSS files that slow down the loading of my page so I find this tool particularly useful. Unfortunately, the downside is that in order to make the CSS smaller, you need to add utility classes to the HTML which can end up making it too long and hard to read. However, I believe with good organisation and differentiation, it can be managed.

Ark UI üì¶

Ark UI is a headless component library that offers accessible and customizable building blocks for modern web applications. It is compatible with any style system and it is also multiframework. Furthermore, it allows developers to have full control over the look and feel since they can create the aesthetic of components which are not tied to specific styles.

Component catalog

The amount of components that Ark UI offers is quite impressive and useful to incorporate in my projects. It includes basic components such as buttons to more complex ones such as passwords, menus and even signature pads or date pickers. It has almost every component you would need for your site so I would highly consider incorporating them in my sites.

The fact that components do not have default styles allows you to customise them in order to fit your aesthetic which is really useful to build a design-centered and original website. Ark UI gives you the base and you can build up from there. It is also compatible with many styling solutions such as TailwindCSS so using both would be an option to consider in future projects.

Static Site Generators and Templates

Hugo üü¢

Hugo is a static site generator written in Go, which is fast and safe perfect for bigger sites with more pages. It organises and transforms content into webpages ready to be published through themes and templates. It is easy to install although it need manual configuration in GitHub.

Organisation

I really like the way Hugo organises content in different sections and folders with hierarchy so everything is organised and well-structured. It saves time and effort and it is effective and reliable.

Learn more
Themes

Hugo applies layouts and themes which saves time and effort. It has themes which combine layouts, CSS and assets ready to use and download and that are customisable.

Learn more
Modules

The use of modules allows you to organise, extend and reuse parts of the website as they were packs as if they were construction blocks. It can be as small as a menu or a collection of images to an entire theme.

Learn more

Jekyll üü•

Jekyll is an open-code static site generator written in Ruby which takes files written in Markdown or HTML, processes them with templates, and converts them into a static website (HTML, CSS, and JS files) that can be easily published. It is a bit slower in the generation of bigger sites but it can be easily integrated with GitHub pages.

Front Matter

I learnt about front matter and the use of variables to manage metadata in Jekyll using YAML. This allows you to establish variables such as title or layout at the beginning of the site in order to use them across different pages. This ensures consistency and it saves time and space since you only need to change it once in the front matter instead of in every individual page.

Learn more
Liquid Template Language

Something that caught my attention about Jekyll is that it uses its own template language called Liquid. This allows you to reuse certain components and elements across different pages of the site which keeps consistency and saves time. I like how it acts as a glue between content (Markdown) and metadata (YAML), placing them into HTML layouts. The result is a static site with dynamic content which I find very interesting and convenient.

Learn more
Pagination

I would not usually use pagination in a website but if I wanted to create my own design blog I think it would be very useful to have automatic pagination so I would definitely consider using this Jekyll tool.

üåü Reflection

I found especially useful to thoroughly analyse different tools and resources in order to incorporate them in my future project, as well as template options as a starting point. The objective is to facilitate the creation of efficient and useful site, taking into account aesthetics and usability.

Speed vs. Flexibility

Speed

  • Pros ‚úÖ: Speed is crucial for a faster charging and publishing webpage which is necessary when you need to achieve results in a short period of time. It is also more accessible to learn as the use of frameworks such as Bootstrap can accelerate the development of the page.
  • Cons ‚ùå: When you prioritise speed you can lack uniqueness and adaptability with the risk of depending on templates which can end up looking too generic. Furthermore, it might be able to maintain if it needs to be scaled or developed later on.

Flexibility

  • Pros ‚úÖ: Flexibility allows your webpage to grow and adapt in the long run so it can expand beyond the prototyping phase. In addition, it makes it easier to create a unique and brand-adapted design which can integrate different tools other than pre-made templates. You also have more control over user¬¥s experience and the web¬¥s performance.
  • Cons ‚ùå: If you choose flexibility you might have to spend more time developing the project as it requires a higher technical level for more complex projects.

Conclusion

There is not right or wrong but it all depends on the kind of project you want to develop. In the particular case of the portfolio, if it is a static page that does not need maintenance I would choose speed. However, I would personally want to incorporate dynamic and time-changing storytelling, so for me flexibility would be essential. Identity, scalability and creative control are preferable so I would opt for a more flexible approach even though it implies spending more time on the development of the project.

Node.js vs. Standalone

Node.js

  • Pros ‚úÖ: Node.js is ideal for certain applications such as chats or online streaming with real-time data. It is compatible with Windows, mac and linux and its JavaScript code also executes very quickly.
  • Cons ‚ùå: Although it is very fast, it can consume memory in large projects and it might be harder to debug and it may also need more to take a more careful look into updates and possible errors.

Standalone

  • Pros ‚úÖ: One of the most obvious pros of using Standalone is its simplicity and reliability which will ultimately take less time to charge.
  • Cons ‚ùå: It takes more manual work to create complex features such as filters, search, loading... and it is limited regarding the inclusion of external services or data.

Conclusion

I think for this project I would use standalone development as the foundation for speed and simplicity but I would also consider integrating tools such as Ark UI in the future, which incorporates logic and accessibility to components in order to add flexibility and creativity. I believe a good balance between both approaches is ideal.

Simplicity vs. Dependencies

Simplicity

  • Pros ‚úÖ: Simplicity means only employing plain HTML, CSS and JS in your project. One of the main advantages is that you have full control of everything that your code does and you don¬¥t depend on external libraries. Additionally, it is more lightweight and faster which result in better performance and SEO. Moreover, it allows you to work on the same project and update it without updates or failures linked to external programs and it gives you more creative freedom without the risk of creating a standard, no personality page.
  • Cons ‚ùå: One of the main disadvantages is the longer initial develop time as you don¬¥t have pre-made components and grids. You need to master HTML, CSS and JS in order to create a full functioning web and it can become a bit chaotic in bigger projects if there is not organisation. You also risk creating inconsistent pages across the web which can be prevented with the use of some dependencies.

Dependencies

  • Pros ‚úÖ: Using dependencies in your projects such as Bootstrap, Tailwind or Material UI, allow us to develop webpages in a faster and more efficient way, incorporating layouts, ready-made components or animations. Furthermore, it generates consistency and cohesiveness, creating professional-looking and uniform websites.
  • Cons ‚ùå: If too many dependencies are used, the project might end up being too heavy and harder to load. In addition, you might lose creativity as you are using pre-made components and elements that might look too standard if not personalised. You also depend on external programs and libraries so you might generate some dependency to them, having to adjust to updates, new versions and even possible errors you cannot

Conclusion

I feel the solution is to incorporate a hybrid method in which there is a solid HTML, CSS and JS code as a base which you can fully control and then add the necessary dependencies such as Tailwind for utilities or Animate.js for animations.

Convention vs. Creativity

Convention implies following established industry patterns such as frameworks or design systems. This could mean using Bootstrap for the layout or pages with a default structure.

Creativity means breaking the rules, experimenting with interaction, fonts and layouts without following trends or standards. You could use Anime.js to add unique microinteractions to a portfolio with a less conventional navigation or visual storytelling.

Convention

  • Pros ‚úÖ: Some of the advantages are familiarity for the user since it imitates standard patterns that repeat across pages. It also becomes faster to develop and its appearance is professional and standardised.
  • Cons ‚ùå: On the other hand, it lacks visual identity as you risk appearing as another template which can make you stand out less in the creative industry, specially if you are creating your portfolio.

Convention

  • Pros ‚úÖ: It is clear that creativity makes you memorable as it reflects your personal or brand identity and makes you innovate in UI and UX with animations, visuals and interactions.
  • Cons ‚ùå: You have the risk of confusing the user who is used to traditional layouts and interfaces, breaking established usability patterns. It also takes more time, effort and technical abilities to develop a working webpage.

Conclusion

There is not a single solution and it varies based on your personal preferences. I would want a creative and personal portfolio but I also like simplicity as a starter point. I would make sure navigation is clear and it follows some of the most important standards. In addition, I would use a legible font for the body of the webpage paired with a funkier and more creative typography for titles or small pieces of information. I would incorporate creativity with the inclusion of Anime.js or GSAP microinteractions, a unique colour palette and visual storytelling where my design pieces speak for themselves.

How would I incorporate these tools in my portfolio site:

    I am not entirely sure I would use these tools for my portfolio or just create my site from scratch but these are some of the options I would consider choosing:

  • Framework base: I would employ Bootstrap due to its consistency and simplicity. The downside of Bootstrap is that it could end up looking too generic which is not the objective.
  • Animations: I would use GSAP for smooth and light animations because it has a bit more complexity than Anime.js but it is still easier than doing the animations myself from scratch.
  • UI Components: I would consider employing Ark UI to organise and add logic and accessibility to Bootstrap components.
  • Static generator: I would consider using Hugo as a static site generator to start with a strong base.
  • Images: For image optimisation I would definitely use ImageKit as I really like its interface and it is crucial to optimise images.
